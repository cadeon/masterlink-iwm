Task Groups
Previous  Top  Next

Description:
   
Tasks Groups are a system capability that allows the building of relationships between Tasks within an Object. These are usually inherited from the template, but may be created or customized for an individual object. They are useful components as they allow the user to indicate which, if any, tasks should be grouped together in a 'single job' when they happen simultaneously.    

Term clarifications:

·Stand-alone is an instance created with no reference to a template.  
 
·Custom is an instance which was created originally from a template and its link to the template was removed as result of modification to an inherited property.
