Action Template

In a similar manner that Tasks within an Object Template are the subcomponents that specify the requirements about the template. Tasks are composed by at least one action that form a list of requirements that must be met for that task. (i.e. to-do list). Actions themselves are one-line instructions of what is to be done for that task. In the field, each Action is accompanied by a space to provide field notes, called "Field Conditions" in the system.  

Actual Time

Refers to the time taken for the completion of a task in the field and reflects uncontrollable or unforeseen circumstances. It may be different from the Estimated Time.  

Analyze

The Analyze section is not currently active. This section will include components that will allow for the analysis of organizational performance via simulation capabilities.  

Button
 
Predefined computer interface object that indicates execution of a command/request upon activation.  
 
Calendar

The Calendar section of IWM is the system capability that allows the automation agents assign work to the most appropriate human resource, based on that resource's availability. Maintaining an accurate calendar will ensure that the system optimizes your organization to its true potential and avoids scheduling mistakes. The Calendar is a critical source of information that the system utilizes to optimally plan and schedule work by matching work to the most appropriate available human resource. The Calendar subsection allows users to show when and where a human resource will be available to work. Thus this information must represent what is actually happening in the organization (i.e. Update the calendar when a person is sick, or changing a work location).  
 
Comment

Found in some Jobs, it is a brief description of any special consideration or explanation of the work to be performed. If the job is a result of a "Maintenance Request" it refers to any details provided by the requester.  
 
Completed Date

Indicates the date in which the job was completed.  

Candidate

Indicates a person / worker who matches the requirements for a job.  

Control

The Control section contains the components that allow the daily monitoring and management of activities through the system. In short they identify, What work needs to be done, Who is doing the work, What stage of completion a series of jobs is in, What feedback has been given from work.  
 
Created By

Indicates the who created an specific item. Ex. The user who created the job one is reviewing.  
 
Created Date

Indicates the date in which the job was created in the system.  
 
Cursor

A movable screen indicator which marks the location where one may activate a function on the computer interface.  

Custom

Indicates that the item (ex. Task) has been changed from the default defined in the Template section. This condition removes the ability inherit changes done to the Template to protect this Custom state. An asterisk next to the number link for an item dictates what is custom.  
 
Days Between Service

Indicates the frequency, in number of days, that a task should be performed (ex. every day, every three days, etc.)  

Define

The Define section contains the components that will define how your organization will be run. In short they define, What needs to be worked on; Who is going to perform the work; Where work is to be performed. Proper setting of this section  

Delete

A function that indicates an item is to be removed permanently from the system. Some items may not be deleted from the system, they may only be deactivated, reconfigured (edited) or canceled (jobs). Usually none-deletable items have been used by the system in the past and cannot be deleted to maintain accurate historical records of the activities managed by the system .  

Description

Used to identify the item in reference (ex. identifies a Task Group Template, Task Group, Task Sequence, Job).  
 
Special Condition: Job  
 
"Description" is a required field that is initially generated by the system using the information given by the Object Task. This field is editable and may be changed to provide persons in the field with further job clues.  
 
Dispatched Date

Indicates the date in which the job was received by a person in their work schedule.  

Earliest Start

Indicates how soon this job may be ready for scheduling.  

Edit

A function that allows users to change an item and its properties in the system.  
 
End Shift

Is a function that allows users to end their daily shift in the system at the end of their working day. If you have ended your shift prematurely see Reset Schedule  
 
Estimated Time

Indicates the estimated time a task or job should consume. It may be edited to adjust for any variations from normal job conditions after analysis of "Actual Time" recordings.  

Field Condition

Is a field in all task actions that provides space for workers in the field to provide feedback.  
 
Special Condition: Meter Reading Job  
 
A "Meter Reading Job" is used to collect age or use information from an object (ex. a Vehicles mileage). The information recorded in the "meter reading" action automatically updates that objects "meter".  
 
Filters

Provide users the capability to search and narrow down onto a more specific item.  
 
Frequency

Indicates the frequency in which a task must take place.  

Special Condition: Task Templates & Tasks  

"Frequency" is a required field ONLY IF the "Task Type" is "Routine Maintenance".  
 
There are two options for choosing a frequency.  
 
1.Select from the preformatted frequencies in the "Frequency" drop-down menu.  
 
2.Specify a custom frequency interval using the Service Interval Fields either by time (Months Between Service, and Days Between Service) or use (Meter Increment).  

Full Locator

Indicates the complete locator as presented in the system's nomenclature (ex. East.02-PAC.In.1st = East Site, 02-PAC Building, Inside, First Floor)  

Home Page

Is the default central screen displayed upon entering the IWM system. It is intended to be a central bridge to all components of the system.  
 
Implement

The Implement section contains the components that allow the activation of the policies established in the Define section. In short they identify, What in specific needs to be work on (instances of items defined in Templates; and What sequential procedures should take place.  

In-Service Date

Indicates when a Locator was created or placed into service in the system.  

Instances

Represent specific occurrences of the more general Template specified in the Define section of IWM.  

Job

Represent a work order intended for a worker matching the requirements for the tasks within it (ex. date, location, skill, skill level).  

Job ID

Is a unique numerical identification generated by the system for each Job.  

Job Status

Indicates the present state of a job.

Last Planned Date

Indicates the last time that task was scheduled.  
 
Special Condition: Routine Tasks Types  
 
The system will use this date to calculate the next time the task is required again.  
 
Latest Start

Indicates at task's expiration date. The system uses this date to understand when a task in no longer relevant and thus no longer schedule it. The task state becomes "expired".  
 
Link

Is a predetermined area on a screen that when activated by a mouse click executes a command to display a new screen.  
 
Locator

Is a location are represented by IWM's location hierarchy.  

Locator Abbreviation

Is a shortened name given to a Locator for display purposes in order to reduce the length of the Full Locator.  

Locator Address

Is the physical address of a Locator, used for reference only.  

Locator Data

Is Data that is related to a particular location (ex. voltage on a room, dimensions of a room, etc.)  

Login

A function that allows users to enter the IWM system in a secure manner, in order to restrict unauthorized use.  

Logout

A function that allows users to exit the IWM system in a secure manner, in order to restrict unauthorized use.  

Maintenance Request

An unscheduled job created through the Maintenance Request system component.  

Measure

The Measure section is not currently active. This section will include components for the tracking of performance in multiple perspectives and summaries (Reports)  
 
Meter Increment

Indicates interval between task execution requirements as measured in life / use units (ex. mileage, hours of use, etc.).  

Special Condition: Routine Tasks Types  
 
The system will use this number to calculate the next meter reading for which the task is required again.  

Modifier

The mediating component of "Action" statements for a task (ex. Verb Name Modifier = Clean System As Required).  

Months Between Service

Indicates the time interval between task execution requirements as measured in months.  

Special Condition: Routine Tasks Types  
 
The system will use this number to calculate the next date for which the task is required again.  

Name

The noun component of "Action" statements for a task (ex. Verb Name Modifier = Clean System As Required).  
 
Number of Workers

Indicates the number of workers required for the completion of a task.  

Object Data

Is Data that is related to a particular object (ex. serial number, color, etc.)  

Object Data Template

Is Data that is related to a particular type of objects (ex. dimensions, fuel type, etc.)  

Object Type

Indicates the category of an object: [image]  
 
·Assessment  
 
·Facilities  
 
·Office  
 
·Production  
 
·Operations Improvement Program  

Organization

Is a representation in the system of a group of individuals who share some overarching commonality. It does not necessarily match the sites organizational structure.  

Organization ID

Is a unique numerical identification generated by the system for each Organization.  

Password

A secret term used to gain access to secure places or information.  

People
 
Identifies individuals entered in the system.  

Person

Identifies individuals entered in the system, may include active or inactive states. See Worker.  

Priority

Indicates how critical a task or job is; it is used for scheduling by the system to prioritize jobs.  

Problem Selector

Identifies the type of problem from a list of predetermined categories when creating a Maintenance Request (unscheduled job).  

Project

Is a group of jobs that must happen in a particular sequence in order to accomplish a goal.  

Project Job

Jobs created for a project. This are managed in a the Project section of IWM and are not shown in the Jobs section of IWM.  

Schedule

A list of jobs assigned to a worker in the system for a particular day (i.e. days assignment list).  

Scheduled By

Indicates who is responsible for scheduling the task once it has been created as a Job. "Manual" scheduling gives the most control but does not allows the system to perform to peak performance when optimizing schedules. "System" allows the system to automatically handle scheduling through its optimization processes.

Security Level

Indicates the clearance level associated with a Location, Object or Task. It is used by the system to restrict workers from obtaining jobs for which they do not have adequate security clearance.  

Service Plan

Indicates when in time should a task be planned (ex. Some task may only be relevant in winter, or during weekends, etc.).  

Shift

Indicates the time period a worker is available for work.  

Skill

Indicates the type of proficiency a worker has, or that is required by a task for completion. This is used by the system to optimally match a worker to a task.  
 
Skill Level

Indicates the level of proficiency a worker has, or that is required by a task for completion. This is used by the system to optimally match a worker to a task.  
 
Start Date

Indicates the date in which a task is available for system monitoring.  

Started Date

Indicates when a job was started (the date when a worker first recorded time for that job).  
 
Status

The present state of a Person, Object, Task, or Job in the system. Persons, Objects and Tasks may be Active.  
 
Tag

Indicates a label given to an object in the system to identify and differentiate from others of the same kind.  
 
Task Description

A label given to briefly identify the nature of the task and its actions (ex. Cleaning Service).  
 
Task Group Template

A group of tasks for a type of object that should be scheduled in a single job when they happen to occur concurrently.  

Task Group

A group of tasks for a specific object (an instance of a template) that should be scheduled in a single job when they happen to occur concurrently.  
 
Task Sequence

A series of Task, potentially from different objects, that must occur in a particular order. Unlike a project whose jobs must be manually created, a Task Sequence creates the jobs automatically.  
 
Task Template

A task belonging to an Object Template. Used to establish a requirement that must be met by all instances of that Object Template.  

Task Type

Indicates the properties of a task. Task Types are:  
 
Routine Maintenance  
 
Assessment  
 
Repair  
 
Replace  
 
[image]  

Template

A set of characteristics that define a type of Objects, Tasks, Actions (a mold) from which multiple individual specific instances are created.  

Tenant Email

The contact information from the initiator of a Maintenance Request, its email.  

Tenant Name

The name of the initiator of a Maintenance Request.  

Tenant Phone

The contact information from the initiator of a Maintenance Request, its phone number.  

Time Recording

The function allowing workers to record time spent on a Job.  

Verb

The active component of "Action" statements for a task (ex. Verb Name Modifier = Clean System As Required).  
 
People
 
Identifies individuals entered in the system.  
 
 
Worker Type

Indicates the type of organizational association a worker has. It also indicates the required association a worker must have to work on a task.
