Description:   

The Implementation section of IWM encloses the instantation components of the system. Within this section "Object Templates" and "Locators" come together to create specific objects (instances of Object Templates) that exist in the client's organization and are used to schedule work. The two subsections of this section are: Objects and Project Stencils.

 

·   The Objects subsection is a repository of object instances that exist in the client's organization. They are represented in the system to allow for accurate resource management and ensure that they receive the attention they require based on the defined requirements in the "Object Templates". Once an instance is created (an Object) it will contain all the requirements imposed by the "Object Template", nonetheless these requirements may be customized to adjust the needs of a single instance or small group of instances (a small enough group not to warrant an its own template). To simplify universal changes any modification made to an "Object's" template is instantly inherited by the object, unless that particular object has been "customized", in which case all ties to the template are broken after conception.

·   Project Stencils are a subsection designed to create repeatable sequences of tasks that may incorporate different objects and resources. Each Project Stencils is labeled and contains within it a list of all the tasks that must be completed for that sequence. In addition it includes a "step" mechanism that allows one to group tasks in chronological clusters. This capability enables the system to enforce the completion of a single or group of tasks, before any other of the subsequent tasks are scheduled.
