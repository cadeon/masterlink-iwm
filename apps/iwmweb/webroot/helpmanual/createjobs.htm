Task:   Creating a Job in IWM.
   
Description:   Allows users to create new Jobs in the system.    

1.Select the "Add Job" button in the Jobs screen.  
 
This will display the "Create Job: Select Object" screen.      
 
 

2.   Filter the hierarchy object class filters to find the object you desire from the "Object" screen.

Locate the Object for which you wish to create a job.  

Note: Available Objects have a "Select" button located to the left of the "Object Name". A highlighted object means that object is not ready for job creation.  

3.   Select the "Select" button for the object.
    then click on the "next button" .
      
This will display the "Create Job: Assign Task" screen.      
 
 
 
4.Assign the Task using the check boxes to the left of the task.      
 
5.Select the "Next" button to confirm. This Will dsplay the "Create Job: Set Details" screen.  
 
 
 
 
Select the "Finish" button to confirm the transaction and edit the job details or the "Cancel" button to discard them.  

Unless edited the new Job will follow the conditions indicated on the Object and Tasks for which it was created.  

6.   If "Edit Button" was selected. Review the conditions from the "Edit Job" screen.

 

You may edit the fields as best suits the present conditions.  

·"Job ID" is a system generated field that uniquely identifies the Job.  
 
·"Created By" is a system generated field which identifies who created the present Job.  
 
·"Created On" indicates the date in which the job was created in the system.  
 
·"Description" is a required field that is initially generated by the system using the information given by the Object Task. This field is editable and may be changed to provide persons in the field with further job clues.  
 
·"Note" is free text space provided for any additional comments that may serve the person performing the job on the field.  
 
·"Number of Workers" indicates the number of skilled persons required to complete this job.  
 
·"Sticky Job" If a job is scheduled manually via (Job schedule screen) it is sticky (Sticky is ON). If a job is unscheduled manually via (Job schedule screen) it is not sticky (Sticky is OFF). If mobile worker opens his schedule (logs in the system) sticky flag is set ON for all jobs assigned to him. If job has sticky flag set, Scheduler will the keep job assignment to the previously assigned person.  
 
·"Est. Time" indicates the estimated time this job should consume. It may be edited to adjust for any variations from normal job conditions.  
 
·"Priority" indicates the urgency for this job.  
 
·"Skill" indicates the type of skill required for this job.  
 
·"Skill Level" indicates the level of a skill proficiency that is required for this job.  
 
·"Job Type" indicates the category of the job.  
 
·"Organization" indicates any organizational restrictions for who may perform this job. Used for when multiple organizations share the same skill.  
 
·"Schedule By" indicates who is responsible for scheduling the task once it has been created as a Job. "Manual" scheduling gives the most control but does not allows the system to perform to peak performance when optimizing schedules. "System" allows the system to automatically handle scheduling through its optimization processes.  
 
·"Status" indicates the present job's status in the system.  
 
·"Earliest Start Date" indicates how soon this job may be ready for scheduling.  
 
·"Latest Start Date" indicates the expiration date for this job, after which the job will no longer be relevant or needed.
